Detailed Description

This file contains functions that perform database operations on the data search tables. Many of the functions use the db ops wrappers provided by the DIRT Core module.

◆ dirt_datatable_get_all_data_from_data_table()

dirt_datatable_get_all_data_from_data_table (   $max_access_level)

Returns entire data table formatted for data search table page, i.e.,

  • rows with matching site_id and survey date (month+year) are combined
  • nid and survey_type columns are hidden
  • only rows with sharing_permission <= max_access_level returned.

Note that, as an exception to the rule above, all participants will be able to view their own data in the search table while logged in (even if their data is semiprivate).

Parameters
[in]$max_access_levelMax access level integer determined by logged-in user's access (0, 1, or 2).
Return values
ArrayAn associative array indexed first by row number and second by column name, e.g. $table_rows[$row_number]['survey_month'] = 6.

◆ dirt_datatable_update_data_table_for_uid()

dirt_datatable_update_data_table_for_uid (   $uid)

Update the data table with one user's surveys.

Note: update count might be off if any survey has "other" selected as habitat type because this will update the table to set the habitat type as entered by the user then a second update process for site desc surveys will add back in "Other: " prefix for habitat type (required formatting for data search table). This could be fixed in the future by adding the "Other: " prefix to habitat type when the survey is saved, though would need to be careful of hitting database storage limits if prefix + text >255 characters.

Parameters
[in]$uidUser ID.
Return values
Arraycontaining keys: 'inserted' => Number of rows inserted into the table, 'updated' => Number of rows updated in the table, 'deleted' => Number of rows deleted from the table. Returns empty array if an error occurred along the way.
